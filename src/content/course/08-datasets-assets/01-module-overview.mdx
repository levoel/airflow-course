# Datasets → Assets — обзор модуля

Data-aware scheduling — революция в DAG dependency model. Вместо «запусти каждый день в 02:00» можно сказать «запусти когда asset orders_table обновлён». Этот модуль покрывает evolution Datasets (2.4) → Assets (3.0) и расширения: Asset Partitions (3.2), Event-driven AIP-82.

## Ключевые концепты

```python
# 2.4-2.11 (Datasets):
from airflow import Dataset
my_dataset = Dataset("s3://bucket/data.parquet")

@dag(schedule=[my_dataset])
def consumer(): ...

@dag(schedule="@daily")
def producer():
    @task(outlets=[my_dataset])
    def write(): ...

# 3.0+ (Assets):
from airflow.sdk import asset

@asset(schedule="@daily")
def orders_asset(context):
    return df  # автоматически outlet

@dag(schedule=[orders_asset])
def downstream(): ...
```

## Asset Partitions (3.2)

Downstream может подписаться на конкретную partition asset-а, не на весь asset:

```python
@dag(schedule=[orders_asset.partitions("dt=2026-05-12")])
def daily_processor(): ...
```

## Event-driven (AIP-82)

Внешние события через message bus триггерят DAG:

```python
from airflow.providers.common.messaging import CommonMessageQueue

queue = CommonMessageQueue(
    queue_url='sqs://my-queue',
    schema='orders_v1'
)

@dag(schedule=queue)
def process_orders(): ...
```

Поддерживаются: AWS SQS, Apache Kafka, Google Pub/Sub.

## Killer takeaway

Data-aware scheduling решает фундаментальную проблему cron: DAG-A запускается в 02:00, DAG-B зависит от A, запускается в 02:30. Если A задерживается до 02:35 — B бежит на старых данных. С Assets B запускается только когда A эмитит asset event, независимо от времени.
